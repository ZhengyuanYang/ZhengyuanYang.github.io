Expectation Maximization Algorithm(EM)

1. Baisc Idea of Expectation Maximization Algorithm

The Expectation Maximization Algorithm(EM) is a type of algorithm sharing the same basic idea. It’s not a specific model but a way to solve problems like parameter estimation and clustering.

First, recall the formula of the Maximum Likelihood Estimation. Given a sample set, we suppose the variable follows a distribution P(X∣θ)P(X|\theta)P(X∣θ). According to MLE, we can estimate the parameter θ\thetaθ by:

θ=argmax  ln[∏nP(Xn=x∣θ)]=argmax  ∑nln[P(Xn=x∣θ)]\theta = argmax \ \ ln[\prod_n P(X_n = x|\theta)] = argmax \ \ \sum_n ln[P(X_n = x|\theta)] θ=argmax  ln[n∏​P(Xn​=x∣θ)]=argmax  n∑​ln[P(Xn​=x∣θ)]

Suppose we now have a hidden variable ZZZ that has j possible values. We cannot directly read Z from the dataset. Under each possible value ziz_izi​, X follows a unique distribution X∼D(θ=θi)X \sim D(\theta = \theta_i)X∼D(θ=θi​). The Likelihood estimation function should now be:

(θ,Z)=argmax  ∑nln[∑iP(Xn=x,Zn=zi)∣θi)](\theta, Z) =argmax \ \ \sum_n ln[\sum_i P(X_n = x,Z_n = z_i)|\theta_i)] (θ,Z)=argmax  n∑​ln[i∑​P(Xn​=x,Zn​=zi​)∣θi​)]

For example, suppose we want to estimate the average height of a group of student. As we all known, the average height of male and female are different. Hm∼N(μ1,σ12),Hf∼N(μ2,σ22)H_m \sim N(\mu_1,\sigma^2_1),H_f \sim N(\mu_2,\sigma^2_2)Hm​∼N(μ1​,σ12​),Hf​∼N(μ2​,σ22​). Suppose we do not have the variable sex in our data set, but we do know there are two distribution, and we want to estimate both of them. In this case, when given a sample, there are actually two thing to do:

  1. What’s the gender(hidden variabl) of a sample?
  2. Given the gender, update according distribution’s average height(parameter)

We can see when we need a EM algorithm: There exists multiple distributions. We don’t know which distribution a sample is drawn from, but we do know this process is decided by a hidden variable not collected in the data.

Now back to the equation of above, we know that finding out the derivatives of this function is difficult, as the log of the margin distribution log(∑zP(x,z))log(\sum_z P( x, z))log(∑z​P(x,z))'s derivative is very hard to solve. In this context, we can transform the format:

∑nln[∑iP(Xn=x,Zn=zi)∣θi)]=∑nln[∑iQ(zi)P(x,zi∣θi)Q(zi)]\sum_n ln[\sum_i P(X_n = x,Z_n = z_i)|\theta_i)] = \sum_n ln[\sum_i Q(z_i)\frac{P(x,z_i|\theta_i)}{Q(z_i)}] n∑​ln[i∑​P(Xn​=x,Zn​=zi​)∣θi​)]=n∑​ln[i∑​Q(zi​)Q(zi​)P(x,zi​∣θi​)​]

Where we demand Q(z)Q(z)Q(z) is a new distribution such that: ∑Q(zi)\sum Q(z_i)∑Q(zi​)

according to the Jensen inequality, log(∑iλiyi)≥∑iλi[log(yi)]log(\sum_i\lambda_iy_i) \ge \sum_i\lambda_i[log(y_i)]log(∑i​λi​yi​)≥∑i​λi​[log(yi​)] , the equality is satisfied when yi=cy_i = cyi​=c.

Thus:

ι(θ,z∣X)=∑nln[∑iQ(zi)P(x,zi∣θi)Q(zi)]≥∑n∑iQ(zi)logP(x,zi∣θi)Q(zi)\iota(\theta,z|X) = \sum_n ln[\sum_i Q(z_i)\frac{P(x,z_i|\theta_i)}{Q(z_i)}] \ge \sum_n\sum_iQ(z_i)log\frac{P(x,z_i|\theta_i)}{Q(z_i)} ι(θ,z∣X)=n∑​ln[i∑​Q(zi​)Q(zi​)P(x,zi​∣θi​)​]≥n∑​i∑​Q(zi​)logQ(zi​)P(x,zi​∣θi​)​

According the condition of the equality fulfillment of Jensen inequality:

P(x,zi∣θi)Q(zi)=cP(x,zi∣θi)=cQ(zi)∑iP(x,zi∣θi)=c∑iQ(zi)=cQ(zi)=P(x,zi∣θ)c=P(x,zi∣θi)∑iP(x,zi∣θi)=P(x,zi∣θi)P(x,∣θi)=P(zi∣x,θi)\begin{aligned} \frac{P(x,z_i|\theta_i)}{Q(z_i)} &= c \\ P(x,z_i|\theta_i)&= c Q(z_i)\\ \sum_i P(x,z_i|\theta_i)&= c \sum_i Q(z_i) = c\\ Q(z_i) &= \frac{P(x,z_i|\theta)}{c} = \frac{P(x,z_i|\theta_i)}{\sum_iP(x,z_i|\theta_i)}\\ &= \frac{P(x,z_i|\theta_i)}{P(x,|\theta_i)} = P(z_i|x,\theta_i)\\ \end{aligned} Q(zi​)P(x,zi​∣θi​)​P(x,zi​∣θi​)i∑​P(x,zi​∣θi​)Q(zi​)​=c=cQ(zi​)=ci∑​Q(zi​)=c=cP(x,zi​∣θ)​=∑i​P(x,zi​∣θi​)P(x,zi​∣θi​)​=P(x,∣θi​)P(x,zi​∣θi​)​=P(zi​∣x,θi​)​

We now know that when we select Q(zi)=P(zi∣x,θi)Q(z_i) = P(z_i|x,\theta_i)Q(zi​)=P(zi​∣x,θi​), what we get is a low boundary of the log likelihood function. We know that:

P(zi∣x,θi)=P(zi,x∣θi)P(x∣θi)=P(zi,x∣θi)∑iP(zi,x∣θi)P(z_i|x,\theta_i) = \frac{P(z_i,x|\theta_i)}{P(x|\theta_i)} = \frac{P(z_i,x|\theta_i)}{\sum_iP(z_i,x|\theta_i)} P(zi​∣x,θi​)=P(x∣θi​)P(zi​,x∣θi​)​=∑i​P(zi​,x∣θi​)P(zi​,x∣θi​)​

Suppose the value of the hidden variable is ziz_izi​, and the parameter of the distribution when Z=ziZ = z_iZ=zi​ is θi\theta_iθi​, the basic assumption of the problem can be solved by EM is, when θi\theta_iθi​ is fixed for a value, and the sample results x are given, we should be able to calculate the specific value of P(zi,x∣θ=θi)P(z_i,x|\theta = \theta_i)P(zi​,x∣θ=θi​)

Such a fact means, we can directly calculate Q(Z)Q(Z)Q(Z) from the observation on data. This save us from calculating the Z∗Z^*Z∗ by ∂L∂Z=0\frac{\partial L}{\partial Z} = 0∂Z∂L​=0, where Z∗Z^*Z∗ denotes the optimal value of ZZZ about L when θi\theta_iθi​ is fixed.

According to such assumption, the EM algorithm can be conclude as:

  1. Randomly select a value for each θi\theta_iθi​ in the first iteration, denoted as θi,0\theta_{i,0}θi,0​

  2. Expectation: For each sample xnx_nxn​, calculate the posterior probability Qn(zi)=P(Zn=zi,Xn=xn∣θi=θi,0)Q_n(z_i) = P(Z_n = z_i,X_n = x_n|\theta_i = \theta_{i,0})Qn​(zi​)=P(Zn​=zi​,Xn​=xn​∣θi​=θi,0​). This step should yield a constant Qn(zi)Q_n(z_i)Qn​(zi​) for each ziz_izi​ for each sample n.

  3. Maximization: bring Q(z)Q(z)Q(z) back the likelihood function, since Q(zi)Q(z_i)Q(zi​) is now a constant and do not have an influence to derivative, we can reformat the likelihood function as:

    θi,1=argmax  ∑n∑iQn(zi)log(P(xn,Zn=zi∣θi,0))\theta_{i,1} = argmax \ \ \sum_n\sum_i Q_n(z_i)log(P(x_n,Z_n = z_i|\theta_{i,0})) θi,1​=argmax  n∑​i∑​Qn​(zi​)log(P(xn​,Zn​=zi​∣θi,0​))

  4. Let θi=θi,1\theta_i = \theta_{i,1}θi​=θi,1​, repeat step 2-4, until θ\thetaθ converge

2. Example of EM

Suppose we have 2 coin A and B, each with a different probability p1,p2p_1,p_2p1​,p2​ of flipped as a head. We randomly select a coin and flip it 10 times and call it a trail. We repeat 5 trail and obtian 5 samples. Let:

suppose we have the following data(unlike shown in the picture, we do not know which coin is selected). Suppose the prior knowledge P(Z=A)=12P(Z =A) = \frac{1}{2}P(Z=A)=21​ is fixed, and we do not want to update it.

Step 1

Initialize θA=0.6, θB=0.5\theta_A = 0.6, \ \theta_ B = 0.5θA​=0.6, θB​=0.5

Step 2

P(Z1=A∣X1,θA,1)=12P(Z1=A,X1=5∣θA,1=0.6)12P(Z1=A,X1=5∣θA,1=0.6)+12P(Z1=B,X1=5∣θB,1=0.5)=0.65∗(1−0.6)50.65∗(1−0.6)5+0.55∗(1−0.5)5=0.45\begin{aligned} P(Z_1 = A|X_1,\theta_{A,1}) &= \frac{\frac{1}{2} P(Z_1 = A, X_1=5|\theta_{A,1} = 0.6)}{\frac{1}{2}P(Z_1 = A, X_1=5|\theta_{A,1} = 0.6) + \frac{1}{2}P(Z_1 = B, X_1=5|\theta_{B,1} = 0.5)} \\ & =\frac{0.6^5*(1-0.6)^5}{0.6^5*(1-0.6)^5 + 0.5^5*(1-0.5)^5} \\ & = 0.45 \end{aligned} P(Z1​=A∣X1​,θA,1​)​=21​P(Z1​=A,X1​=5∣θA,1​=0.6)+21​P(Z1​=B,X1​=5∣θB,1​=0.5)21​P(Z1​=A,X1​=5∣θA,1​=0.6)​=0.65∗(1−0.6)5+0.55∗(1−0.5)50.65∗(1−0.6)5​=0.45​
